How this works under the hood:
1. Design Time — AI reads the objective and generates an input schema (required + optional fields, permission types)
2. Run Time — UI renders only the fields that are required or that the user hasn't skipped
3. Required fields block the Run button. Optional fields default to sensible values or can be skipped.
4. Tokens/keys are never saved — used for the session only, cleared on close.
